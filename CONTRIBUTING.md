# Guide de contribution

## Workflow Git

Le projet utilise Git Flow.

- `main` : branche stable
- `develop` : branche de développement
- `feature/*` : branche pour chaque fonctionnalité

## Étapes de contribution

1. Créer une issue
2. Créer une branche depuis `develop`
3. Faire les modifications
4. Faire un commit clair
5. Push la branche
6. Créer une Pull Request vers `develop`
7. Relier la PR à l'issue avec `Closes #numéro`
